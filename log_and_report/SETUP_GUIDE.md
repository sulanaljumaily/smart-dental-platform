# دليل تفعيل تطبيق SMART الطبي (Backend Setup)

## نظرة عامة
هذا الدليل يوضح كيفية ربط النظام بقاعدة بيانات Supabase الحقيقية.

## المتطلبات
1. حساب على Supabase.
2. مشروع جديد (New Project).

## الخطوة 1: إعداد متغيرات البيئة (Environment Variables)
المشروع يستخدم Vite، لذا يجب أن تبدأ المتغيرات بـ `VITE_`.

1. أنشئ ملف `.env.local` في المجلد الرئيسي.
2. أضف البيانات التالية من إعدادات مشروعك في Supabase:

```env
VITE_SUPABASE_URL=YOUR_PROJECT_URL
VITE_SUPABASE_ANON_KEY=YOUR_ANON_KEY
```

> تجد هذه المعلومات في: Settings > API

## الخطوة 2: إنشاء الجداول (Database Schema)
بما أن قاعدة البيانات فارغة، نحتاج لإنشاء الجداول الخاصة بالنظام الطبي.

1. اذهب إلى **SQL Editor** في لوحة تحكم Supabase.
2. افتح الملف `supabase/schema_doctor.sql` الموجود في هذا المشروع.
3. انسخ محتواه الكامل وضعه في SQL Editor ثم اضغط **Run**.
   - هذا سينشئ جداول: `clinics`, `staff`, `patients`, `appointments`, `treatments` وسياسات الأمان (RLS) الخاصة بها.

4. (اختياري) إذا كنت بحاجة لجداول المتاجر والوظائف، انسخ محتوى `supabase/schema.sql` وشغله أيضاً.

## الخطوة 3: التحقق والتشغيل
1. أعد تشغيل السيرفر المحلي:
   ```bash
   npm run dev
   ```
2. سيقوم النظام باكتشاف المتغيرات وسيعمل في الوضع الحقيقي (Real Mode) بدلاً من التجريبي (Mock Mode).
3. **ملاحظة هامة**: المستخدمون التجريبيون (مثل `staff-101`) لن يكونوا موجودين في القاعدة الحقيقية.
   - يجب عليك **إنشاء حساب جديد** عبر صفحة التسجيل في التطبيق.
   - أو إضافتهم يدوياً في Supabase Authentication وجدول `profiles`.

## استكشاف الأخطاء
- **شاشة بيضاء / خطأ اتصال**: تأكد من أن الرابط يبدأ بـ `https://` وأنه صحيح.
- **البيانات لا تظهر (RLS)**: اذا قمت بتسجيل الدخول ولم تظهر بيانات، تأكد من أن المستخدم مرتبط بعيادة (Clinic ID) في جدول `staff` أو أنه المالك (`owner_id`) في جدول `clinics`.
